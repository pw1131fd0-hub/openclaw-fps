## 1. 產品願景與核心痛點（Why）

### 1.1 願景
打造一款「打開瀏覽器就能玩」的輕量級第一人稱射擊遊戲，讓玩家無需下載安裝，即可享受流暢的 FPS 體驗。

### 1.2 核心痛點
- **下載門檻高**：傳統 FPS 遊戲需要數 GB 下載、安裝，對於想「隨便玩一下」的用戶來說太麻煩
- **設備限制**：辦公室電腦、學校電腦無法安裝遊戲，但有瀏覽器
- **跨平台困難**：想在手機、平板也能玩同一款遊戲
- **離線需求**：通勤、飛機上也想玩，但沒有網路

### 1.3 解決方案
OpenClaw FPS 是一款基於 Three.js/WebGL 的網頁 FPS 遊戲：
- 純前端、靜態部署，打開網址即玩
- 支援 PWA，可離線遊玩、加到手機主畫面
- 低多邊形美術風格，效能友好
- 波次生存模式，簡單上手、容易沉浸

## 2. 目標用戶畫像與使用場景

### 2.1 目標用戶
| 用戶類型 | 特徵 | 需求 |
|---------|------|------|
| 休閒玩家 | 25-40 歲上班族，午休或下班想放鬆 | 快速上手、不需要學習複雜操作 |
| 學生族群 | 15-22 歲，課間休息時間有限 | 能在學校電腦上玩、不需安裝 |
| 通勤族 | 每天花 30+ 分鐘在通勤 | 手機可玩、離線可用 |
| 輕度遊戲愛好者 | 不想投入太多時間在遊戲上 | 單機模式、無社交壓力 |

### 2.2 User Stories

#### US-01: 午休快速開局
> **作為**一名上班族  
> **我想要**在午休時打開瀏覽器就能玩 FPS  
> **以便**在 15-20 分鐘內放鬆一下

**驗收標準：**
- 首次載入 < 5 秒
- 無需註冊/登入即可開始遊戲
- 按 ESC 可暫停，不會丟失進度

#### US-02: 手機上的 FPS 體驗
> **作為**一名通勤族  
> **我想要**在手機上用觸控方式玩 FPS  
> **以便**在捷運/公車上打發時間

**驗收標準：**
- 觸控搖桿控制移動
- 觸控螢幕瞄準射擊
- 支援 PWA 離線遊玩
- 可加到手機主畫面

#### US-03: 挑戰高分紀錄
> **作為**一名追求成就的玩家  
> **我想要**看到自己的最高分紀錄  
> **以便**有動力持續挑戰更高波次

**驗收標準：**
- 本地儲存最高分
- HUD 顯示當前分數和波次
- 遊戲結束畫面顯示本次成績與最高分對比

#### US-04: 多樣化武器體驗
> **作為**一名 FPS 愛好者  
> **我想要**使用不同武器對付敵人  
> **以便**有策略性和多樣性的遊戲體驗

**驗收標準：**
- 3 種武器可選：手槍、散彈槍、突擊步槍
- 按 1/2/3 快速切換
- 每種武器有不同的傷害、射速、彈匣特性

## 3. 功能需求清單

### 3.1 P0 - 核心功能（MVP 必須）

| ID | 功能名稱 | 描述 | 驗收標準 |
|----|---------|------|----------|
| P0-01 | 第一人稱視角控制 | WASD 移動、滑鼠瞄準、空白鍵跳躍 | 移動流暢、無明顯延遲；跳躍有重力感 |
| P0-02 | 射擊系統 | 滑鼠左鍵射擊、準星置中 | 射擊有即時回饋、hitmarker 顯示擊中 |
| P0-03 | 武器系統 | 手槍/散彈槍/突擊步槍，按 1/2/3 切換 | 切換有動畫、每種武器特性不同 |
| P0-04 | 敵人 AI | 敵人巡邏、發現玩家追擊、攻擊 | 近戰敵人衝向玩家、遠程敵人會射擊 |
| P0-05 | 波次生存模式 | 每波敵人數量和強度遞增 | 第一波 3 隻敵人，每波增加 2 隻 |
| P0-06 | 血量系統 | 玩家有血量，被攻擊會扣血 | 血量歸零遊戲結束 |
| P0-07 | 彈藥系統 | 武器有彈藥限制，需要換彈 | 彈匣空時按 R 換彈 |
| P0-08 | HUD 介面 | 顯示血量、彈藥、波次、分數 | 左下血量條、右下彈藥、上方波次/分數 |
| P0-09 | 遊戲地圖 | 封閉競技場，有掩體、高台、走廊 | 地圖大小適中，約 50x50 單位 |
| P0-10 | 補給拾取 | 血包、彈藥包可拾取 | 靠近自動拾取，有視覺/音效回饋 |

### 3.2 P1 - 進階功能

| ID | 功能名稱 | 描述 | 驗收標準 |
|----|---------|------|----------|
| P1-01 | 排行榜 | 本地儲存最高分紀錄 | 顯示歷史最高分 Top 5 |
| P1-02 | 音效系統 | 射擊、換彈、敵人死亡、受傷音效 | 所有關鍵動作有音效回饋 |
| P1-03 | 多張地圖 | 倉庫、廢墟、太空站 | 3 張地圖可選 |
| P1-04 | 武器升級 | 每 5 波可選擇升級（傷害/射速/彈匣） | 升級選單清晰、效果明確 |
| P1-05 | PWA 支援 | 可離線遊玩、加到主畫面 | 離線時仍可正常遊玩 |
| P1-06 | 觸控控制 | 手機虛擬搖桿和射擊按鈕 | 觸控操作流暢、不卡頓 |

### 3.3 P2 - 未來功能

| ID | 功能名稱 | 描述 |
|----|---------|------|
| P2-01 | 多人連線模式 | WebRTC P2P 對戰 |
| P2-02 | 更多武器 | 狙擊槍、火箭筒、手榴彈 |
| P2-03 | Boss 戰 | 每 10 波出現 Boss |
| P2-04 | 成就系統 | 解鎖成就獲得獎勵 |
| P2-05 | 自訂角色外觀 | 換裝、換色 |

## 4. 非功能需求

### 4.1 效能需求
| 指標 | 目標值 | 量測方式 |
|------|--------|----------|
| 幀率 (FPS) | ≥ 60 FPS 穩定 | 中階電腦 (i5 + 8GB RAM) Chrome |
| 首次載入時間 | < 5 秒 | Lighthouse Performance |
| 打包大小 | < 5 MB (gzip) | Vite build output |
| 記憶體使用 | < 512 MB | Chrome DevTools Memory |

### 4.2 相容性需求
| 環境 | 最低要求 |
|------|----------|
| 桌面瀏覽器 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 行動裝置 | iOS 14+ Safari, Android 10+ Chrome |
| 螢幕解析度 | 最低 320x568 (iPhone SE), 最佳 1920x1080 |

### 4.3 可用性需求
- 首次進入有簡單操作教學
- 暫停選單可查看操作說明
- 支援鍵盤 + 滑鼠、觸控兩種控制模式

### 4.4 安全需求
- 無後端，無用戶資料上傳
- LocalStorage 僅儲存遊戲設定和分數
- 不使用任何第三方追蹤或廣告 SDK

### 4.5 擴展性需求
- 模組化架構，方便新增武器/敵人/地圖
- 設定檔驅動，武器參數可調整
- 資源載入器支援動態載入新地圖

## 5. 技術限制與選型

### 5.1 技術棧

| 層級 | 技術選型 | 理由 |
|------|----------|------|
| 渲染引擎 | Three.js (WebGL) | 成熟穩定、社群大、文檔豐富 |
| 物理引擎 | Cannon-es (Cannon.js 維護版) | 輕量、適合簡單碰撞偵測 |
| 語言 | TypeScript | 型別安全、更好的開發體驗 |
| 打包工具 | Vite | 快速 HMR、原生 ESM 支援 |
| 樣式 | CSS3 (原生) | HUD 簡單，不需要框架 |
| 部署 | 靜態檔案 (Netlify/Vercel/GitHub Pages) | 無後端、簡單部署 |

## 6. 競品分析

| 競品 | 優點 | 缺點 | 我們的差異化 |
|------|------|------|-------------|
| Krunker.io | 成熟、多人連線 | 需要帳號、廣告多 | 單機、無廣告、即開即玩 |
| Shell Shockers | 有趣的蛋形象 | 多人依賴、延遲問題 | 離線可玩、PWA 支援 |
| 1v1.LOL | 建造機制獨特 | 操作複雜 | 簡單上手、專注射擊 |
| Zombs Royale | 大逃殺模式 | 需要網路、俯視角 | 第一人稱沉浸感 |

## 9. 成功指標（KPI）

| 指標 | 目標 | 量測方式 |

## 10. 附錄

### 10.1 術語表
| 術語 | 定義 |
|------|------|
| HUD | Head-Up Display，遊戲內即時資訊介面 |
| Hitmarker | 擊中敵人時的視覺回饋（準星變化） |
| Wave | 波次，一群敵人的攻擊單位 |
| PWA | Progressive Web App，漸進式網頁應用 |
| Low-poly | 低多邊形風格，使用少量多邊形的美術風格 |
